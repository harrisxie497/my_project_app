# 日本清关/派送 Excel 自动处理系统

## 项目概述

本项目是一个基于 Vue3 + Element Plus 的前端应用，用于实现日本清关和派送 Excel 文件的自动处理。

## 项目结构

```
├── src/
│   ├── App.vue          # 主应用组件
│   └── main.js          # 应用入口
├── index.html           # HTML 模板
├── package.json         # 项目配置和依赖
├── vite.config.js       # Vite 配置
└── README.md            # 项目说明
```

## 功能模块

### 1. 用户登录
- 支持管理员和操作员两种角色
- 基于 JWT 的认证机制

### 2. 任务中心
- 上传 Excel 文件（清关文件或派送文件）
- 查看任务列表和详情
- 运行任务
- 下载处理结果

### 3. 规则配置
- 字段映射规则管理
- 模板映射管理
- AI 能力配置
- 用户管理
- 操作日志查看

## 技术栈

- **前端框架**: Vue 3
- **UI 组件库**: Element Plus
- **构建工具**: Vite
- **包管理器**: npm

## 安装和运行

### 安装依赖

```bash
npm install
```

### 开发模式

```bash
npm run dev
```

### 构建生产版本

```bash
npm run build
```

### 预览生产版本

```bash
npm run preview
```

## API 接口

项目使用 RESTful API 与后端通信，主要接口包括：

- 认证相关：登录、获取当前用户
- 任务相关：创建任务、运行任务、查询任务列表和详情、下载文件
- 规则配置相关：管理规则表、规则项、模板映射、AI 能力配置
- 用户管理相关：创建用户、启用/禁用用户、重置密码
- 操作日志相关：查询日志

## 注意事项

- 本项目目前为前端原型，无真实后端实现
- 登录功能使用模拟数据
- 任务处理和文件下载功能为模拟实现
- 管理员角色可以访问所有功能，操作员角色只能访问任务中心

## 开发说明

### 代码风格

- 使用 Vue 3 的 Composition API
- 采用 `<script setup>` 语法糖
- 组件样式使用 `<style scoped>`

### 状态管理

- 目前使用组件内部状态管理
- 可根据需求扩展为 Pinia 或 Vuex

### 路由管理

- 目前使用组件内部路由（简单的条件渲染）
- 可根据需求扩展为 Vue Router
